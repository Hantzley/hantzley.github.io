1 - Lab Overview

Amazon Q Business application is a generative AI assistant that boosts employee productivity and transforms the way that you get work done. You can quickly find accurate answers from your enterprise content, backed up with citations and references. You can brainstorm new ideas, generate content, or create summaries using Amazon Q application based on your enterprise data. Amazon Q application ensures that users access enterprise content securely according to their permissions. You can quickly deploy Amazon Q application with built-in connectors to popular enterprise repositories.


Solution Overview

In this lab, we will build a technical documentation assistant using Amazon Q Business to help developers find and answer questions about Swagger APIs, also known as the OpenAPI Specification, which provides a standardized way to describe APIs.

In this situation, the Swagger API documentation consists of JSON or YAML files, flow diagrams, and other non-rich text formats. Amazon Q Business does not handle these data types by default, therefore we will need to run an enrichment process using this notebook to generate synthetic documentation based on YAML files and images for Amazon Q for Business. 3 - Setting up document repository

Amazon Q Business rely on your enterprise's content. In this lab, we will create a content repository on Amazon S3. However, Amazon Q can pull your enterprise data from a range of sources, for example, Amazon RDS, Box, Salesforce, Google Drive, etc.

Follow the steps below to create a document repository in Amazon S3:

  • Open Amazon S3 console.
  • Click the Create bucket button.

  • Give your bucket a name. This is in a global name space, so if your bucket name is already taken, please type in a different one.

  • Leave all other options by default and click the Create bucket button.

  • Your new bucket is now created and should appear in the S3 bucket list.

  • You will need to gather some documents to upload in the repository. You can use your own documents for your business use case if you have it at hand. For convenience, you can use the following documents:
    • AMZIBEE Employee Handbook
    • Ministry of Man Power - Guide on Employment Law
    • Amazon Leadership Principles
  • Download the documents on local machine.
  • To upload the documents in your S3 bucket, select your S3 bucket and then click the Upload button.

  • Add all files, then click Upload button at the bottom of the page.

  • You can click on the Close button.

4 - Creating an Amazon Q Application

  • Amazon Q Business applications needs an identity provider, in our case, identity federation and management takes place in AWS identity Center.
  • Before creating the Amazon Q Business application, let's disable MFA in IAM identity Center to make it easier to onboard users for this lab.
  • Go to IAM identity Center console .
  • If it is the first time accessing IAM identity Center for this region, click Enable button.

  • In the dilog window, select Enable in only this AWS account option, then click Continue button, and Enable button.

  • Select Settings in the menu options on the left.
  • For MFA Settings, select Never, then Save changes.

  • Now, let's go to the Amazon Q Business console to create the application.
  • Click on the Create application button.

  • Give your application a name.
  • You will notice that the application is connected to an account instance of IAM Identity Center.
  • Click Create button.

  • Select Use native retriever for this lab.
  • If your organization already has an Amazon Kendra retriever, you can use it.
  • Click Next

  • Amazon Q Business allows you to connect multiple data sources in a single application. However, for this lab, we will use the document repository on S3 that you already created.
  • Click the + sign next to Amazon S3

  • Give the data source a name.

  • For IAM role, click on the dropdown menu and select Create a new service role (Recommended).

  • Under Sync scope, click on the Browse S3 button.

  • In the popup dialog, select your document repository S3 bucket.

  • For Sync mode, select Full sync.
  • For Sync run schedule, click on the dropdown menu. Then select Run on demand

  • At the bottom of the page, select Add data source.

  • You will notice next to Amazon S3, that one data source is added.

  • At this point, you can add more Amazon S3 buckets as data source. Or explore other connectors that Amazon Q Business supports.
  • When done, click the Next button at the bottom of the page.

  • Now we create new users within Amazon Q Business. These users will show up under the IAM Identity Center console as well.
  • Click the Add groups and users button.

  • The popup dialog allows us to add and assign new users or assign existing users and groups.
  • Since we don't have any users in IAM Identity Center yet, select Add and assign new users option and click Next button.

  • On the next dialog, click Add new users button.

  • Fill in the user details.
  • Make sure you use an email address to which you have access to for this lab.

  • You can add more users at this stage. But for this lab, we will add only one user.
  • Click Add button.

  • Click Assign button.

  • The user will be listed under Users section.
  • Change the Current subscription to Q Business Pro.
  • Click Create application button.

  • Access the user email. You should see an email with an invation to join AWS IAM Identity Center.
  • Click the Accept invitation button in the email.

  • Set a new password for the user.

  • Now login with the new credentials

  • After loging in, you will see the Amazon Q application in the list. Click on the application to launch it.

  • At this stage you will not be able to query your data as we need to sync the data source first.

7 - Advanced options

  • By default, the application knowledge is limmited to the information in your data source.
  • For example, if you ask a question which is not found in your data source, Amazon Q will tell you that it could not find relevant information for your request.

  • We can configure Amazon Q to fall back to the LLM knowledge if relevant information is not in your data source.
  • Open the Amazon Q Business console and click on Admin controls and guardrails option in the menu on the left.

  • Click on the Edit button in the Global controls section.

  • Check the Allow Amazon Q to fall back to LLM knowledge option and click Save button.
  • Let's try the same question again.

  • You will notice that now, Amazon Q will let you know that no answer was found in your data source, but will generate based on what it knows.
  • You can also configure Amazon Q to allow users to send queries directly to LLM.
  • Check the Allow end users to send queries directly to the LLM option in Admin controls and guardrails settings and save.
  • Then, in the Amazon Q application, disable the Respond from approved sources control:

  • You will notice under the prompt input box, the indication All data sources off.
  • Now you can ask any general question outside the context of your data source.


  • Amazon Q also allows you to customize the UI for your organization.
  • Open the Amazon Q Business console , select your application and click on Customize web experience button.

  • You can change the Title, Subtitle and Welcome message.
  • You can also check the Display sample prompts option if you wish.

  • Close this window (or tab) and refresh the Amazon Q application.
  • You should see the customizations.

9 - Creating Apps within Amazon Q

Amazon B Business gives your end users with Pro subscription, the ability to create their own apps and publish them in the main Amazon Q application's library for others to use.

  • On the Amazon Q application, select the App icon on the vertical menu bar on the left.
  • Click on the Interview Question Generator option. This will populate the prompt required to generate an end user App. Then click the Generate button.
  • This application has file input widgets for job description and resume. Upload a job description file and a resume. Then click the Run button.

  • After testing your App. You can publish it so that others can find it in the Apps library.
  • Click on the Publish button.
  • Select relevant labels.
  • Then click on the Publish button.

  • Go to the Apps library. The App should appear with the respective labels.

10 - Setting up Amazon Q Plugin (Jira)

Amazon Q Business allows you to strigger actions using plugins. We will use Jira plugin in this lab to trigger a leaves request for our HR App.

  • If you don't have an Atlassian account, go to Atlassian website and create a trial account.
  • After logging in, click on the Apps icon (top left) and select Jira.
  • Create a project.

  • Click on your user icon (top right)
  • Select Manage account

  • Select Security and Create and manage API Tokens

  • Click Create API Token button.
  • Give it a label.
  • Copy and save the API token. We will use later.

  • Now go to the Plugins section in the Amazon Q Business console .
  • Click Add plugin button.

  • Available plugins are shown here.
  • Click the + button next to the Jira plugin.

  • Give the plugin a name.
  • Key in the base URL of your Jira cloud instance.
  • Under AWS Secrets Manager secrets, click the dropdown menu and select Create and add new secret

  • Give the secret a name.
  • Key in your username. This is typically the email address you used to register an account on Atlassian.
  • Paste your API token in the API token input box.

  • Leave the options as default.
  • Click Add plugin button.

  • After creation, it should show up as active in the plugin list.

  • Typically, you will use the plugin in a conversation. For example, in our HR App, after asking about parental leave, you can trigger the plugin. Amazon Q will use the conversation context then triggering the plugin.
  • To use the plugin in the Amazon Q application, click the icon with 3 dots next to the prompt and select the Jira plugin.

  • Type in the a request that will be sent to the Jira plugin and submit.

  • Amazon Q will show up a form for submitting a Jira issue. You will notice that the description is pre-populated with the context of your query and prior conversation.
  • Select your project.
  • Fill in the Reporter Email and click the Submit button.

  • Amazon Q will display the response from Jira, or an error message if it could not communicate with Jira.

  • In your project on Jira, the request should show up.
